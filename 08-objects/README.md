# Lesson 08 – Objects

## Overview

This lesson introduces JavaScript objects and demonstrates how to organize related data using key-value pairs. It covers creating objects, accessing and updating properties, adding and deleting properties, and iterating through object properties using a `for...in` loop.

---

## Concepts Covered

- Object Creation
- Properties
- Key-Value Pairs
- Dot Notation
- Bracket Notation
- Updating Properties
- Adding Properties
- Deleting Properties
- `for...in` Loop

## Practice Exercise

Create a JavaScript program to complete the following tasks.

### Exercise 1

Create a `student` object with the following properties:

- name
- age
- department
- cgpa

Print every property.

---

### Exercise 2

Update the student's `cgpa`.

Print the updated object.

---

### Exercise 3

Add a new property:

```text
city
```

Print the updated object.

---

### Exercise 4

Delete the `city` property.

Print the object again.

---

### Exercise 5

Create a `car` object with:

- brand
- model
- price

Use a `for...in` loop to print every property and its value.

## Mini Project

### Student Profile Manager

Create a JavaScript program to manage a student's profile using an object.

Create a student object with:

- Name
- Age
- Department
- College
- CGPA
- IsHosteller

Perform the following operations:

1. Display all student details.
2. Update the CGPA.
3. Add a new property called `phoneNumber`.
4. Delete the `phoneNumber` property.
5. Display the final student object using a `for...in` loop.

**Note:** Use only the concepts learned so far. Do not use `Object.keys()`, `Object.values()`, `Object.entries()`, destructuring, or spread syntax.

## Learning Outcomes

After completing this lesson, I can:

- Create JavaScript objects.
- Store related information using key-value pairs.
- Access object properties using dot and bracket notation.
- Modify object properties.
- Add and remove properties.
- Iterate through object properties using a `for...in` loop.

---

## Common Mistakes

- Using the wrong property name.
- Forgetting that property names are case-sensitive.
- Accessing properties that do not exist.
- Confusing arrays with objects.
- Using bracket notation incorrectly.
